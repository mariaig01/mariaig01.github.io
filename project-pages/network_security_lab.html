Network Security Monitoring Lab
In Development — Network Security
A home lab environment for capturing and analysing network traffic, detecting intrusions in real time, and centralising security logs into a SIEM dashboard. The goal is to practise the same monitoring workflows used by SOC analysts in real organisations.
What This Project Is About
The idea behind this project is to build a small but realistic security monitoring setup using virtual machines. Instead of just reading about how network monitoring works, the goal is to actually set it up, generate traffic, and see alerts fire in real time.
The lab will use VirtualBox or Proxmox to run a few VMs on a local network. One machine will act as a sensor running Suricata, an open-source intrusion detection system (IDS) that inspects network packets and raises alerts when it spots suspicious patterns — like port scans, brute-force attempts, or known exploit signatures.
Wireshark will be used alongside Suricata to manually inspect captured traffic — looking at packet headers, following TCP streams, and understanding what "normal" vs. "suspicious" traffic actually looks like at the packet level.
All the logs and alerts from Suricata will be forwarded to a Wazuh + ELK (Elasticsearch, Logstash, Kibana) stack. Wazuh acts as the SIEM — it collects logs from multiple sources, correlates events, and presents everything in Kibana dashboards. This is the same kind of setup many companies use for security monitoring, just scaled down to a home lab.
What I'll Learn
How to capture and analyse network traffic at the packet level. How IDS rules work and how to write custom Suricata rules. How to set up a SIEM pipeline — from raw logs to searchable, visual dashboards. How to distinguish between normal network noise and genuinely suspicious activity. Practical experience with the tools SOC analysts use daily.
Technologies
Wireshark • Suricata • Wazuh • Elasticsearch • Logstash • Kibana • VirtualBox / Proxmox • Linux
